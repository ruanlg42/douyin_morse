# 星途信使限时云 UI 与交互设计

## 目标

所有可站立云都具有清晰的三秒停留时限。玩家落地后需要在云消散前完成蓄力并起跳，云体本身持续表达剩余时间，避免只依赖独立数字提示。

## 交互规则

- 玩家落到普通云后立即启动固定 `3.0s` 倒计时。
- 倒计时期间允许正常按压、拖动瞄准和蓄力，时间不会因开始按压而暂停。
- 玩家松手起跳即视为及时离开，当前云停止倒计时，并在身后自然散去。
- 倒计时归零仍未起跳时，当前云破碎消散，角色进入现有坠落与复活流程。
- 起点云在游戏进入 `playing` 状态后启动倒计时。
- 终点信箱云和天门落地即完成，不启动倒计时。
- 暂停、菜单和结算状态不推进倒计时。
- 原有流云不再使用独立时长和重复计时 UI；所有云遵循同一三秒规则。

## 视觉反馈

- `3.0s → 1.2s`：云体保持可读，实体感、阴影和高光缓慢减弱，边缘雾气逐渐逸散。
- `1.2s → 0.8s`：云体轮廓开始松散，计时环与数字明显增强。
- `< 0.8s`：进入警戒态，使用克制的珊瑚色边缘脉冲和轻微抖散，不闪烁全屏。
- `0s`：云团向两侧分离并快速淡出，角色自然坠落。
- 时间环放在云体右上侧，避开点划符号、尖刺提示和安全落点区域。
- 点云、划云和通用云继续保留金、蓝、白三套语义色，警戒色只在最后阶段出现。

## 状态设计

- 游戏状态保存统一的 `cloudTimer`，当前站立云保存 `lifeRemaining`、`lifeTotal` 和 `spent`。
- 落地和起点初始化统一调用云计时激活逻辑。
- 起跳统一清理计时，并将离开的云标记为 `spent`，只影响后续视觉，不影响空中碰撞。
- 超时处理具备幂等保护，避免同一帧重复触发坠落、扣命或提示。

## 验证

- 验证起点、普通云、任务云、尖刺云、双频云均在落地后从三秒开始计时。
- 验证按住蓄力超过三秒会导致云消散和坠落。
- 验证三秒内松手起跳不会被上一朵云的超时状态误伤。
- 验证目标云、点划符号和尖刺安全区在渐隐期间仍清晰可辨。
- 验证暂停状态不消耗计时。
- 运行现有物理与关卡码单测、生产构建，并进行浏览器实机测试。
